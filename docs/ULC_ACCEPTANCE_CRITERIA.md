# ULC MVP Acceptance Criteria

The first version is acceptable only if it satisfies these criteria.

## Product and Methodology

1. The product is not a PDF viewer.
2. Lessons are native interactive web pages.
3. The MVP is limited to Beginner English for Adults.
4. Content is original ULC content.
5. Methodological logic is comparable to English File / New English File, but no copyrighted text, dialogue, exercise, image, audio, character, workbook page, or page design is copied.
6. The learning model is speaking-first.
7. Every lesson has a practical speaking outcome.
8. Beginner interface and task instructions are in Russian.
9. English learning content remains in English with Russian support where needed.

## Student Experience

10. Student Dashboard exists.
11. Student can see course, level, unit, lesson, homework, and progress.
12. Student has a clear next action.
13. Student can use native lesson pages.
14. Student can complete homework mode.
15. Student can see progress and certificate progress.
16. Student can review vocabulary or weak areas.

## Exercise and Feedback

17. Exercise Player exists.
18. Required MVP exercise types are represented: fill in the gaps, multiple choice, matching, drag-and-drop, word order, listen and choose, listen and type, and short writing.
19. Exercises support answer -> auto-check -> correct answer -> explanation.
20. Accepted answer logic supports reasonable Beginner variants such as contractions, case, spacing, and punctuation tolerance where appropriate.
21. AI mistake explanations are available in Russian.
22. AI explanations are short, beginner-friendly, and tied to the current lesson.
23. The external speaking widget is not rebuilt, but the product has a clear integration point for it.

## Teacher and Live Lesson Mode

24. Teacher Dashboard exists.
25. Teacher can see groups.
26. Teacher can see individual students.
27. Teacher can see homework completion.
28. Teacher can see exercise results and typical mistakes.
29. Live Lesson Mode exists.
30. Live Lesson Mode shows simulated or real real-time student answer visibility.
31. Live Lesson Mode includes a shared learning space concept.
32. A two-cursor or co-presence mockup is present for the MVP.

## Methodologist and Content

33. Methodologist Dashboard exists.
34. Methodologist can see course, unit, lesson, section, and exercise structure.
35. Methodologist can see or edit exercise metadata.
36. Teacher book notes are represented.
37. Content status supports draft, review, approved, and archived states.

## Analytics and Integrations

38. Progress tracking exists.
39. Analytics include NPS, retention, homework completion, and depth of usage concepts.
40. Risk signals are represented, including inactivity and missed homework.
41. The architecture is API-ready for 1C, Revizor/BI, Bitrix24, Telegram, payments, scheduling, feedback/NPS, and the external speaking widget.
42. Mock data is acceptable only when real APIs are unavailable, and integration boundaries remain clear.

## UI Quality

43. UI looks like a ULC learning product, not a generic template.
44. Main accent color is `#3C4AF5`.
45. Student homework is mobile-friendly.
46. Teacher and methodologist dashboards are desktop-friendly.
47. Exercise text, inputs, buttons, and feedback do not overlap on mobile or desktop.
48. The interface avoids visual noise and keeps the next action clear.

## Completion Rule

49. A feature is complete only when it can be tested in the browser or verified through a clear non-visual check.
50. Any implementation task must be checked against this document before it is considered done.
